# Architecture Decision Records

This directory contains ADRs for significant decisions made in this plugin's design.

An ADR is warranted when a decision:

- Is hard to reverse (skill structure, naming conventions, distribution model)
- Has non-obvious trade-offs worth preserving context on
- Was actively debated before being settled

For small, obvious, or easily-reversed decisions, a commit message is sufficient.

## Creating an ADR

Copy `0000-template.md`, increment the number, and give it a short hyphenated title:

```text
docs/adr/0001-skill-format-shared-across-platforms.md
```
